<objective>
Fix GitHub issue #5: anchor copy-paste regression where FQNN is not updated when an anchor is moved between scripts.

Purpose: When an anchor node is copy-pasted from script A to script B, the anchor's stored FQNN (in the KNOB_NAME knob) still references script A. This means subsequent copy-paste of link dots pointing to this anchor in script B will fail to reconnect because the FQNN stem comparison detects a "cross-script" mismatch even though the anchor is now local. The fix is to update the anchor's FQNN after a cross-script paste so it reflects its new home script.

Output: Patched anchors.py with regression test.
</objective>

Before the `continue`, add FQNN rewrite logic:

```python
if not input_node:
    # Cross-script case: anchor stays as placeholder.
    # Update the stored FQNN to reflect the destination script so that
    # subsequent copy-paste of links referencing this anchor will see
    # a same-script FQNN and reconnect correctly (GitHub issue #5).
    stored_fqnn = node[KNOB_NAME].getText()
    if stored_fqnn:
        # Replace the source script stem with the destination script stem.
        # FQNN format: "<script_stem>.<node.fullName()>"
        fqnn_parts = stored_fqnn.split('.', 1)
        if len(fqnn_parts) == 2:
            destination_stem = nuke.root().name().split('.')[0]
            node[KNOB_NAME].setValue(f"{destination_stem}.{fqnn_parts[1]}")
    continue
```

This preserves the node.fullName() portion (which may include Group paths like `Group1.Anchor_MyFootage`) and only replaces the script stem.

Note: The `node.fullName()` part may have changed after paste (Nuke auto-renames to avoid conflicts). A more robust approach would be to recompute the FQNN entirely using `get_fully_qualified_node_name(node)`. However, for anchor nodes the fullName includes the Anchor_ prefix which is already correct. Use `get_fully_qualified_node_name(node)` directly — it reads the current script stem and the node's current fullName, which is exactly what we want:

```python
if not input_node:
    # Cross-script: update stored FQNN to destination script (GitHub #5).
    node[KNOB_NAME].setValue(get_fully_qualified_node_name(node))
    continue
```

This single-line fix is cleaner and handles all edge cases (Group nesting, auto-renamed nodes).
