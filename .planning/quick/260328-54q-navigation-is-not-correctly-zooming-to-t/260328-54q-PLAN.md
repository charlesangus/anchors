  <action>
Modify `navigate_to_anchor()` (lines 655-673) to compute a zoom level that fits the bounding box of all nodes_to_fit.

The algorithm:
1. Compute the bounding box of nodes_to_fit:
   - `min_x = min(n.xpos() for n in nodes_to_fit)`
   - `max_x = max(n.xpos() + n.screenWidth() for n in nodes_to_fit)`
   - `min_y = min(n.ypos() for n in nodes_to_fit)`
   - `max_y = max(n.ypos() + n.screenHeight() for n in nodes_to_fit)`
   - `bounding_box_width = max_x - min_x`
   - `bounding_box_height = max_y - min_y`

2. Compute zoom to fit. Use a reference viewport size that represents a typical Nuke DAG panel. A DAG panel at zoom=1.0 shows roughly 1200x800 DAG units (conservative estimate with padding). Add padding (e.g., 20% margin) so nodes aren't at the very edge:
   - `DAG_VIEWPORT_WIDTH = 1200`
   - `DAG_VIEWPORT_HEIGHT = 800`
   - `PADDING_FACTOR = 0.8` (usable area is 80% of viewport)
   - `zoom_to_fit_width = (DAG_VIEWPORT_WIDTH * PADDING_FACTOR) / max(bounding_box_width, 1)`
   - `zoom_to_fit_height = (DAG_VIEWPORT_HEIGHT * PADDING_FACTOR) / max(bounding_box_height, 1)`
   - `zoom_level = min(1.0, zoom_to_fit_width, zoom_to_fit_height)`

3. Keep centroid calculation unchanged.

4. Call `nuke.zoom(zoom_level, [center_x, center_y])` instead of `nuke.zoom(1.0, ...)`.

Define the viewport reference constants at module level near navigate_to_anchor, or as local constants inside the function. Use clear names:
- `_DAG_REFERENCE_WIDTH = 1200`
- `_DAG_REFERENCE_HEIGHT = 800`
- `_DAG_FIT_PADDING = 0.8`

The zoom calculation ensures:
- Small trees (bbox fits within reference viewport): zoom stays at 1.0 (capped)
- Large trees: zoom reduces proportionally so the full extent is visible
- The 0.8 padding factor leaves margin so nodes aren't at the viewport edge
