# Quick Task 260328-54q: Navigate-to-Anchor Bounding-Box Zoom Summary

**One-liner:** Bounding-box-aware zoom in `navigate_to_anchor()` using viewport reference constants (1200x800, 0.8 padding), capped at 1.0 to prevent over-zoom.

## What Was Done

`navigate_to_anchor()` in `anchor.py` previously called `nuke.zoom(1.0, ...)` unconditionally. For large upstream trees this left the user at 1:1 zoom, unable to see the full tree extent.

The fix:
1. Compute the bounding box of all `nodes_to_fit` (anchor + upstream nodes).
2. Derive `zoom_to_fit_width` and `zoom_to_fit_height` from usable viewport (reference size × padding factor) divided by bounding box dimensions.
3. Set `zoom_level = min(1.0, zoom_to_fit_width, zoom_to_fit_height)` — zoom out for large trees, never zoom in beyond 1.0 for small trees.

Three module-level constants were added near the function:
- `_DAG_REFERENCE_WIDTH = 1200`
- `_DAG_REFERENCE_HEIGHT = 800`
- `_DAG_FIT_PADDING = 0.8`
