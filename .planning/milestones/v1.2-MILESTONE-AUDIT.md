## Tech Debt

### Phase 11 — CI/CD Pipeline

- **Live GitHub Actions run not triggered.** The workflow is locally verified (trigger pattern, test gate ordering, ZIP manifest, release action), but a real `git push origin v1.2.0` has not been executed. CI-01 and CI-02 are marked satisfied based on local structural verification. A live run would confirm production behavior.
- **ZIP artifact not yet downloaded.** `unzip -l paste_hidden-vX.Y.Z.zip` from a real release would fully close CI-02.

### Phase 10 — Code Quality Sweep

- **Nuke runtime callback smoke test pending.** `menu.py` F401 is suppressed via `pyproject.toml` because Nuke string-eval callbacks cannot be traced statically. A live Nuke session confirming each menu action still works would close this gap. One human verification item in `10-VERIFICATION.md`.
- **paste_hidden() C901 deferred.** `copy_hidden` and `paste_hidden` suppress complexity warnings with `noqa: C901`. Structural refactoring was explicitly deferred to protect recent BUG-01/BUG-02 fixes; appropriate future work for a cleanup milestone.

### Phase 8 — Test Infrastructure

- **`unittest discover` without `-t .` UX hazard.** Running `python3 -m unittest discover -s tests` (without `-t .`) fails 43 tests because `tests/__init__.py` is bypassed. The CI workflow uses `pytest tests/` (correct). Consider adding `testpaths = ["tests"]` to `[tool.pytest.ini_options]` in `pyproject.toml` to make `pytest` the unambiguous local invocation as well.
