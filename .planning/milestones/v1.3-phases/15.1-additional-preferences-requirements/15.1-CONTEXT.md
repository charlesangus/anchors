# Phase 15.1: Additional Preferences Requirements - Context

**Gathered:** 2026-03-16
**Status:** Ready for planning
**Source:** User-provided requirements

<domain>
## Phase Boundary

Phase 15 delivered the Anchor Naming section in PrefsDialog with three fields (Regex, Template, Test filename), a live match/no-match validity indicator, and a Reset button. Phase 15.1 extends this with six additional capabilities: a live rendered preview of the template output, an "Advanced" collapsible section hiding the technical fields, demo filename persistence, undoable Reset, and a Publish button that writes settings to a site config path.

</domain>

<decisions>
## Implementation Decisions

### Req 1 — Live preview field
Show the actual template-substituted output (the anchor name that would be generated) against the demo filename. Distinct from the validity indicator (which shows Match/No match/Invalid) — the preview shows the *result string*. Updates live as user types in Regex, Template, or Test filename fields. When regex doesn't match or is invalid, show empty/blank. When template is blank, show the full match (group 0). Located visually near the test filename field.

### Req 2 — "Advanced" collapsible section
The Regex, Template, Test filename, and live preview fields are hidden inside a collapsible "Advanced" section. Collapsed by default (or remembers state). The section header is clickable to expand/collapse. When collapsed, only the section header is visible. When expanded, all three fields + preview + Reset + validity indicator are visible. Implementation: Qt's standard approach — a QGroupBox with checkable header, or a clickable label toggling a container widget's visibility.

### Req 3 — Persist demo filename to prefs
The test filename field currently hardcodes "plate_v003.exr". Add `naming_demo_filename` to prefs.py (default `"plate_v003.exr"`), load/save it with the other naming fields, and seed the dialog from it. Saved on OK (following the working-copy pattern).

### Req 4 — Reset is undoable
After clicking Reset (which blanks Regex and Template), the action can be undone to restore the previous values. Implementation: when Reset is clicked, save the pre-reset Regex + Template values; then Ctrl+Z or an "Undo" action restores them. The cleanest approach for this dialog is to use Qt's QUndoStack with a custom UndoReset command, or simply track a `_pre_reset_snapshot` and connect Ctrl+Z to a restore method. Undo only needs to cover the Reset action (not every keystroke — QLineEdit has its own built-in per-field undo for typed text).

### Req 5 — Publish button
A "Publish" button in the dialog writes the current settings (all naming fields, not just the Advanced ones) to the prefs file at the path specified by an env var. The env var name is `ANCHORS_SITE_CONFIG` (consistent with Phase 16 Site Config). This is for site admins to publish their current naming config to the shared site config file.

### Req 6 — Publish greyed out when env var is not set
The Publish button is disabled (greyed out) when `ANCHORS_SITE_CONFIG` is not set or is empty. No tooltip needed. Checked at dialog open time; does not need to re-check dynamically during the session.

### Claude's Discretion
- Exact visual placement of Publish button (likely in a row below the Advanced section or at bottom of Anchor Naming section)
- Whether "Advanced" collapsed state is persisted to prefs (not required)
- Exact Qt widget used for collapsible section (QGroupBox with flat style + toggle, or custom)
- Whether Publish writes ALL prefs or only naming fields (reasonable default: all current prefs, consistent with how save() works)

</decisions>

<specifics>
## Specific Ideas

- `naming_demo_filename` in prefs.py with default `"plate_v003.exr"` — mirrors `naming_regex` / `naming_template` pattern
- Advanced collapsible: simplest working approach is a `QGroupBox("Advanced")` with `setCheckable(True)` — Qt natively collapses child widgets when unchecked — but this uses a checkbox not a click on the label. Alternative: a QPushButton with `▶ Advanced` / `▼ Advanced` text toggling a QWidget container's `setVisible()`.
- Undo for Reset: track `_pre_reset_naming_snapshot` tuple `(regex, template)` in `_on_reset_naming`; add `_on_undo_reset_naming` method; install an event filter or use QShortcut for Ctrl+Z scoped to the dialog that calls `_on_undo_reset_naming` when a snapshot exists
- Publish writes via `prefs_module.save()` after temporarily setting a different output path — or better, calls a new `prefs_module.publish(path)` function that writes to the given path without touching the default prefs file
- `ANCHORS_SITE_CONFIG` checked via `os.environ.get("ANCHORS_SITE_CONFIG", "")` at `__init__` time; stored as `self._publish_path`; button disabled if falsy

</specifics>

<deferred>
## Deferred Ideas

- Per-session Advanced expanded/collapsed state persistence to prefs
- Dynamic re-check of `ANCHORS_SITE_CONFIG` during dialog lifetime
- Undo for individual keystrokes in Regex/Template (QLineEdit handles this natively)

</deferred>
