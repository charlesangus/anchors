**Correction policy (from CONTEXT.md — apply in strict order):**

**Tier 1 — Stub divergence:** If a FAIL line indicates a stub method returns a wrong type or value compared to real Nuke behavior:
- Fix `tests/stubs.py` to match real behavior
- Run `python3 -m pytest tests/ -x -q` immediately after each fix
- If pytest stays green: correction is complete for this item

**Tier 2 — Production bug exposed:** If the Tier 1 stub fix causes `pytest tests/` to go red:
- The red test reveals the stub was papering over a real production bug
- Fix the production source file (`paste_hidden.py`, `link.py`, or `anchor.py`) to handle the real API behavior correctly
- Re-run `python3 -m pytest tests/ -x -q` — must return green before proceeding

**Tier 3 — Test assertion correction (LAST RESORT ONLY):** If real Nuke behavior is confirmed correct AND the stub now matches AND the test still fails because it was testing the wrong thing:
- Fix only the specific test assertion that tests incorrect behavior
- Document why in the git commit message

**Highest-risk corrections likely needed (from RESEARCH.md):**

1. **nuke.toNode("preferences") returns None in stub** — If validate_stub_alignment.py FAIL shows `toNode("preferences")` is not None in real Nuke, update `tests/stubs.py` `make_stub_nuke_module()`:
   - Change `stub.toNode = MagicMock(return_value=None)` to a callable that returns None for arbitrary nodes BUT returns a MagicMock node for `"preferences"` (so `find_node_default_color()` does not crash)
   - Example fix: `stub.toNode = MagicMock(side_effect=lambda name: MagicMock() if name == "preferences" else None)`

2. **tile_color.value() returns float not int** — If validate_stub_alignment.py FAIL shows `tile_color.value()` returns float in real Nuke:
   - Update `StubKnob.value()` to coerce numeric values: `return float(self._value) if isinstance(self._value, (int, float)) else self._value`
   - Check if any production comparison uses `== 0` with a color int (should use `not value` or `int(value) == 0` instead)

3. **nuke.root().name() default in headless** — If validate_stub_alignment.py shows root.name() returns '' or 'Root' in headless (not a .nk filename):
   - Update `make_stub_nuke_module()` root_obj to use `root_obj.name.return_value = ''` (or whatever real value was observed) if the discrepancy matters for test correctness
   - Check `paste_hidden.py` stem logic: `nuke.root().name().split('.')[0]` — if root returns '', stem is '' which makes fqnn_stem != '' vacuously trigger cross-script path. If this is a real bug, fix paste_hidden.py to guard for empty root name.

4. **Script encounters unexpected crash** — If either validation script crashes with a traceback (not a FAIL line), report the full traceback. Fix the script itself if it is a scripting error; if it is a Nuke API error, investigate and apply Tier 1/2 corrections as appropriate.

**After all corrections applied:**
Run `python3 -m pytest tests/` (full suite, not just -x) and confirm all tests pass.
