**Section: `## Creating Anchors`**

Write numbered steps with outcome notes for each sub-workflow. Per CONTEXT.md decisions:

3. Anchor already selected: press `A` opens the rename dialog. Include placeholder: `![Rename dialog with current anchor name pre-filled](img/anchor-rename-dialog.png)`
4. Unanchored Dot selected: press `A` promotes it to a Dot anchor — size picker first (Medium/Large), then label prompt. Include placeholder: `![Dot size picker for anchor promotion](img/dot-size-picker.png)`
5. Color picker available in creation and rename dialogs. Include placeholder: `![Color picker in anchor dialog](img/anchor-color-picker.png)`

Style: numbered steps with outcome notes (e.g. "3. Name the anchor — the dialog pre-fills from the file path."). Describe *what* happens, not *why*.

**Section: `## Navigating to Anchors`**

Write numbered steps:
1. Press `Alt+A` to open the fuzzy-search picker listing all anchors with colors. Include placeholder: `![Anchor navigation picker](img/anchor-navigate-picker.png)`
2. Select an anchor — DAG zooms to its location.
3. Press `Alt+Z` to return to the previous DAG position (completes the navigation loop).

**Section: `## Copy and Paste`**

Write numbered steps for the happy path:
1. `Ctrl+C` copies selected nodes — input nodes (Read, Camera, etc.) are automatically converted to hidden-input proxies.
2. `Ctrl+V` pastes with smart reconnection — proxies re-pipe to their original input nodes if they exist in the same script at the same level.
3. Include placeholder: `![Before and after paste showing hidden-input reconnection](img/copy-paste-reconnection.png)`
4. Mention that this behavior can be disabled via Preferences (plugin-enabled toggle). One sentence, no detail.
5. Do NOT cover edge cases (cross-script, old-style paste, LINK_CLASSES passthrough) — those are in README.

**Optional closing section: `## Keyboard Reference`**

Include a summary table of the shortcuts covered in this guide only (A, Alt+A, Alt+Z, Ctrl+C, Ctrl+V). Format as a Markdown table with Shortcut and Action columns. Add a note: "See README.md for the full shortcut list."

Throughout: use NO rationale or design explanations. Describe what happens, not why.
