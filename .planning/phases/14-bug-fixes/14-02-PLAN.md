<objective>
Fix BUG-04: pressing "a" on a Dot node invokes _offer_make_dot_anchor(), which creates a Dot-class anchor instead of a NoOp anchor.

Purpose: anchor_shortcut() has a special elif branch that intercepts single-Dot selections and routes them to _offer_make_dot_anchor(). Removing that branch lets the Dot fall through to the existing elif selected: branch, which calls create_anchor() — the standard NoOp path. This makes Dot nodes behave consistently with all other node types when "a" is pressed. _offer_make_dot_anchor() is retained but marked deprecated per the locked decision.

Output: anchor.py with the Dot elif removed and _offer_make_dot_anchor() deprecated; regression tests in test_dot_type_distinction.py covering all anchor_shortcut() dispatch paths.
</objective>

def anchor_shortcut():
    """If a node is selected, create an anchor from it. Otherwise, pick an anchor to create from."""

        return
    selected = nuke.selectedNodes()
    if len(selected) == 1 and is_anchor(selected[0]):
