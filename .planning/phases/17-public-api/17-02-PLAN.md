    RuntimeError guard implementation detail:
    The `import nuke` at module level will succeed (nuke is always available in Nuke's Python).
    The guard is for the case where someone imports api.py outside Nuke (e.g., in offline test).
    In tests, the nuke stub is in sys.modules. The guard checks: `if 'nuke' not in sys.modules`.
    Implement `_assert_nuke_session` as:
    ```python
    def _assert_nuke_session():
        if 'nuke' not in sys.modules:
            raise RuntimeError('anchors API requires a running Nuke session')
    ```
    This works in tests (where we can pop 'nuke' from sys.modules to simulate absence) and in
    real Nuke (where 'nuke' is always in sys.modules).
