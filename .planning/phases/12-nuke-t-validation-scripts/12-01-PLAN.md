5. **Check 1 — BUG-01: link receives anchor tile_color via setup_link_node()**

```python
def check_bug01_link_receives_anchor_color():
    anchor_color = 0xff0000ff  # red — distinct from ANCHOR_DEFAULT_COLOR purple (0x6f3399ff)
    anchor = nuke.createNode('NoOp')
    link = nuke.createNode('NoOp')
    try:
        anchor['tile_color'].setValue(anchor_color)
        setup_link_node(anchor, link)
        actual = link['tile_color'].value()
        assert int(actual) == anchor_color, (
            f"expected anchor color {hex(anchor_color)}, got {hex(int(actual))} — "
            f"ANCHOR_DEFAULT_COLOR is {hex(ANCHOR_DEFAULT_COLOR)}"
        )
    finally:
        nuke.delete(link)
        nuke.delete(anchor)
```

6. **Check 2 — BUG-02: anchor pasted cross-script stays an anchor**

The goal is to call the production `paste_hidden()` function through a real `nuke.nodePaste()` invocation and confirm the anchor node is not replaced by a link node.

Strategy:
- Create an anchor node (NoOp with ANCHOR_PREFIX name, with KNOB_NAME knob set to a cross-script FQNN)
- Set `nuke.root()['name'].setValue('destScript.nk')` so the current script stem is 'destScript'
- Set the anchor's KNOB_NAME knob value to `'sourceScript.Anchor_TestAnchor'` (stem mismatch = cross-script)
- Copy the node with `nuke.nodeCopy(nukescripts.cut_paste_file())`
- Call `paste_hidden()` which internally calls `nuke.nodePaste()`
- Assert the original anchor node still exists and has not been replaced by a link node (check `nuke.toNode(ANCHOR_PREFIX + 'TestAnchor')` is not None and its KNOB_NAME value starts with ANCHOR_PREFIX or the node's Class() is still 'NoOp' not converted)
- Use `finally` to delete any surviving nodes

Note on BUG-02 implementation: `paste_hidden()` uses `nuke.selectedNodes()` to get nodes to process. The pasted nodes come from `nuke.nodePaste()`. To correctly exercise the code path, do NOT try to replicate paste_hidden() internals. Instead, rely on `nuke.nodePaste(nukescripts.cut_paste_file())` returning the pasted nodes and `paste_hidden()` processing them.

Simpler correct approach for the check: create the anchor, set KNOB_NAME to cross-script FQNN, copy it, then call paste_hidden(). After paste_hidden(), verify no 'NoOp' with a link-class name replaced the anchor. The key BUG-02 assertion from CONTEXT.md: "assert the node stays an anchor (not converted to a link)." Check that no node was created with a non-anchor name where the anchor was.
