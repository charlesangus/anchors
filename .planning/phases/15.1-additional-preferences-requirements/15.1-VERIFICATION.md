### 2. Toggle expand/collapse

**Test:** Click "▶ Advanced". Observe fields. Click "▼ Advanced". Observe again.
**Expected:** Click 1 reveals all fields and changes button to "▼ Advanced". Click 2 hides fields and restores "▶ Advanced".
**Why human:** Qt setVisible toggling is a runtime behavior.

### 3. Live preview updates

**Test:** With Advanced expanded, type `(?P<shot>.+)_v\d+` in Regex. Observe Preview. Then type `{shot}_anchor` in Template. Observe Preview update.
**Expected:** Preview shows `plate_v003` (full match) after Regex. Updates to `plate_v003_anchor` after Template.
**Why human:** Signal-driven live UI update requires runtime typing.

### 4. Undoable Reset

**Test:** Enter values in Regex and Template. Click Reset. Press Ctrl+Z.
**Expected:** Reset clears both fields (Test filename unchanged). Ctrl+Z restores both fields to pre-reset values. Test filename is unaffected throughout.
**Why human:** QShortcut and snapshot-restore requires runtime keyboard interaction.

### 5. Demo filename round-trip

**Test:** Change the Test filename field to something other than the default. Click OK. Reopen PrefsDialog.
**Expected:** Test filename field opens with the value that was saved, not `plate_v003.exr`.
**Why human:** Prefs persistence round-trip in a live session requires two dialog open/close cycles.

### 6. Publish button disabled state

**Test:** Open PrefsDialog without `ANCHORS_SITE_CONFIG` set in the environment.
**Expected:** Publish button is visible but greyed out and non-interactive.
**Why human:** Button enabled state depends on runtime env var.

### 7. Publish button active state and file output

**Test:** Set `ANCHORS_SITE_CONFIG=/tmp/anchors_site.json`, open PrefsDialog, click Publish.
**Expected:** `/tmp/anchors_site.json` is created with JSON content containing `naming_regex`, `naming_template`, `naming_demo_filename`, and other prefs keys. The user's personal prefs file is unchanged.
**Why human:** Requires live env var configuration and filesystem verification.
