<objective>
Create a Group-context-aware helper function and update all nuke.allNodes() call sites in link.py, anchors.py, and labels.py so that plugin operations respect the current Group node's nested DAG.

Purpose: When a user enters a Group node in Nuke, nuke.allNodes() without a group= parameter returns nodes at the root DAG level, not nodes inside the Group. This causes anchor scanning, backdrop detection, and link reconnection to fail silently inside Groups. A single helper function that passes group=nuke.thisGroup() fixes all call sites consistently.

Output: Updated link.py with all_nodes_in_context() helper, updated anchors.py and labels.py call sites, test coverage for Group-context behaviour, updated stubs.py with thisGroup support.
</objective>

```python
def all_nodes_in_context(node_class=None):
    """Return all nodes in the current Group context.

    When the user is inside a Group node, nuke.allNodes() without a group=
    parameter returns root-level nodes. This helper passes
    group=nuke.thisGroup() so that operations always target the DAG the user
    is currently viewing.

    Parameters
    ----------
    node_class : str or None
        Optional Nuke node class filter (e.g. 'BackdropNode'). When provided,
        only nodes of that class are returned.
    """
    current_group = nuke.thisGroup()
    if node_class is not None:
        return nuke.allNodes(node_class, group=current_group)
    return nuke.allNodes(group=current_group)
```
