Fix the A-key link creation inside Group nodes so that the Link node is created inside the Group's nested DAG instead of at root level.

Purpose: UAT Test 2 failed — pressing A inside a navigated Group creates the Link at root instead of inside the Group. def anchor_shortcut():
```python
    """If a node is selected, create an anchor from it. Otherwise, pick an anchor to create from."""
    if not prefs.plugin_enabled:
        return
    hit_group = nuke.lastHitGroup()
    with hit_group:

```

Key change: `hit_group` is captured BEFORE any `with` block, then passed to `select_anchor_and_create()`.

**Part B: Modify `select_anchor_and_create()` to accept a `hit_group` parameter and set it on the plugin before `show()`.**

Change the signature from `def select_anchor_and_create():` to `def select_anchor_and_create(hit_group=None):` (around line 585).

Replace the body with:

```python
def select_anchor_and_create(hit_group=None):
    if not prefs.plugin_enabled:
        return
    if QtWidgets is None:
        return
    if hit_group is None:
        hit_group = nuke.lastHitGroup()
    with hit_group:
        if not all_anchors():
            return
    global _anchor_picker_widget
    if _anchor_picker_widget is not None:
        try:
            _anchor_picker_widget.plugin._hit_group = hit_group
            _anchor_picker_widget.under_cursor()
            _anchor_picker_widget.show()
            _anchor_picker_widget.raise_()
            return
        except RuntimeError:
            _anchor_picker_widget = None
    plugin = AnchorPlugin()
    plugin._hit_group = hit_group
    _anchor_picker_widget = _tabtabtab.TabTabTabWidget(
        plugin, winflags=Qt.FramelessWindowHint
    )
    _anchor_picker_widget.under_cursor()
    _anchor_picker_widget.show()
    _anchor_picker_widget.raise_()
```

Key changes:
- Accepts `hit_group` parameter, falls back to `nuke.lastHitGroup()` if None
- Sets `plugin._hit_group = hit_group` BEFORE calling `show()` (which triggers `get_items()`)
- For the cached widget path, also updates `_anchor_picker_widget.plugin._hit_group = hit_group`

**Part C: Modify `AnchorPlugin.get_items()` to stop capturing `_hit_group`.**

Replace `get_items()` (around line 507) with:

```python
    def get_items(self):

        # show() calls get_items().  We must NOT capture lastHitGroup() here
        # because show() runs after the original menu-callback context has
        # exited, so lastHitGroup() would return root.

```

Key change: `get_items()` READS `self._hit_group` (set externally) instead of WRITING it from `nuke.lastHitGroup()`. Falls back to `nuke.root()` as a safety net if `_hit_group` was never set.
