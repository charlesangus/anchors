<objective>
Fix Alt+A anchor navigation inside Group nodes so that the DAG panel pans/zooms to the selected anchor within the Group's DAG view, not the root DAG.

Purpose: UAT Test 3 failed — items appear correctly in the picker but selecting one does not pan/zoom the Group DAG panel. The root cause is twofold: (1) `AnchorNavigatePlugin.get_items()` overwrites `_hit_group` with stale `lastHitGroup()` (same pattern as Gap 1), and (2) `nuke.zoom()` targets whichever Qt DAG panel has focus, but the picker steals focus and upon closing Qt returns focus to root DAG. The fix defers the zoom call via `QTimer.singleShot(0, ...)` so it runs after the picker closes and Qt focus naturally returns to the correct panel.

Output: Updated anchor.py with pre-captured group context for navigation and deferred zoom execution.
</objective>

```python
    def get_items(self):
        # _hit_group is set externally by select_anchor_and_navigate() BEFORE
        # show() calls get_items().  We must NOT capture lastHitGroup() here
        # because show() runs after the original menu-callback context has
        # exited, so lastHitGroup() would return root.
        hit_group = self._hit_group or nuke.root()
        with hit_group:
            items = [
                {
                    'menuobj': anchor_node,
                    'menupath': 'Anchors/' + anchor_display_name(anchor_node),
                }
                for anchor_node in all_anchors()
            ]
            for backdrop_node in nuke.allNodes('BackdropNode'):
                label = backdrop_node['label'].value().strip()

                        'menuobj': backdrop_node,
                        'menupath': 'Backdrops/' + label,
                    })
        return items
```

Key change: Reads `self._hit_group` without overwriting it. Falls back to `nuke.root()` as safety.

**Part C: Modify `AnchorNavigatePlugin.invoke()` to defer navigation via `QTimer.singleShot(0, ...)`.**

The problem: `invoke()` is called from `TabTabTabWidget.create()` (tabtabtab.py line 702), BEFORE `self.close()` (line 704). At invocation time the picker widget still has Qt focus, so `nuke.zoom()` targets whichever panel Qt thinks is active (not the Group DAG). After `close()`, Qt returns focus to the previously-active widget, but by then `nuke.zoom()` has already fired at the wrong panel.

The fix: defer the actual navigation until after the picker closes by using `QTimer.singleShot(0, callback)`. The zero-millisecond timer fires on the next event loop iteration, which is after `close()` has returned and Qt has restored focus to the DAG panel.

Replace `invoke()` (around line 708) with:

```python
    def invoke(self, thing):
        node = thing['menuobj']
        hit_group = self._hit_group or nuke.root()

        def _deferred_navigate():
            with hit_group:
                if not nuke.exists(node.name()):
                    return
                _save_dag_position()
                if node.Class() == 'BackdropNode':
                    navigate_to_backdrop(node)
                    return
                navigate_to_anchor(node)

        # Defer navigation until after the picker widget closes and Qt
        # restores focus to the DAG panel.  nuke.zoom() targets whichever
        # panel has Qt focus, so calling it while the picker is still open
        # would zoom the wrong panel (or do nothing).
        QtCore.QTimer.singleShot(0, _deferred_navigate)
```

Key change: The entire navigation logic (including `nuke.zoom()` inside `navigate_to_anchor`/`navigate_to_backdrop`) is wrapped in a closure and deferred with `QTimer.singleShot(0, ...)`. This ensures the picker has closed and Qt focus has returned to the Group DAG panel before any zoom call executes.
